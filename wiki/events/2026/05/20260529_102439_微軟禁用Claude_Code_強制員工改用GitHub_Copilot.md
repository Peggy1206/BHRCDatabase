# 微軟禁用Claude Code，強制員工改用GitHub Copilot

**Category**: Events
**Tags**: 
**Ingested**: 2026-05-29 10:24 UTC

---

## Summary

微軟高層要求員工在2026年6月30日前停用Anthropic的Claude Code開發工具，改用自家GitHub Copilot CLI。此舉表面理由是工具整合，實質可能涉及成本控制，同時反映Anthropic近期漲價爭議對企業採購決策的影響。

## Structured Analysis

## 事件背景
- 微軟Windows、Microsoft 365等部門執行副總裁Rajesh Jha發出內部公開信
- 要求工程師在2026年6月30日前移除Claude Code，改用GitHub Copilot CLI
- 命令涵蓋微軟多個核心部門

## 核心問題
### Claude Code的受歡迎程度
- Claude Code在微軟員工中人氣超越自家GitHub Copilot
- 使用者不限工程師，連非技術人員也在使用
- 被形容為「可能太受歡迎了」

### 官方理由
- 工具串整合（Toolchain Unification）
- GitHub Copilot CLI與公司GitHub程式庫直接整合
- 強調Claude Code是學習工具，但Copilot CLI更符合企業需求

### 隱含理由
- 成本削減是媒體分析的主要推測
- 將工具整合於公司控制的軟體以降低成本

## Anthropic成本問題背景
### 隱性漲價爭議
- 2026年4月Anthropic推出Claude Opus 4.7
- 官方價格表未變，但更換底層分詞器（Tokenizer）
- 相同文字可能多消耗高達35%的Token

### Claude Code定價爭議
- 定價頁面一度顯示Pro方案（$20/月）排除在Claude Code適用對象
- 用戶可能被迫升級至Max方案（$100+/月）
- 引發開發人員強烈反彈
- Anthropic後來澄清此為小型測試，僅涉及約2%新用戶

## 微軟與Anthropic的複雜關係
- 微軟仍允許員工透過GitHub Copilot使用Anthropic模型
- 消費版Copilot和Microsoft 365底層使用Claude模型
- 此舉幫助微軟減少對OpenAI的依賴
- 表明微軟與Anthropic合作日漸深厚，但在企業內部工具選擇上有所保留

## Deepening Questions

- 這個決策對獵才業務有什麼啟示？企業在選擇AI工具時，成本因素是否會成為未來主要考量，進而影響對AI人才的需求？
- Anthropic的漲價爭議是否會成為產業趨勢？其他AI公司是否會面臨類似的成本壓力，進而改變企業對AI工具的採購策略？
- 微軟這個決策反映了什麼樣的企業文化衝突——員工偏好的工具vs.公司戰略控制？在招聘高級工程師時，工具自由度是否會成為競爭力因素？
- 微軟與Anthropic的關係如何演變？這是否預示著大型科技公司會逐步建立自主的AI工具生態，減少對外部供應商的依賴？
